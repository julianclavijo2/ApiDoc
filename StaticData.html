Static Data

All static data, especially hotel information must be stored in the local server and should be updated at least once a month. To get all static data from our hotels and cities inventory, you must consume the rest services with the URL found on each section, resulting a JSON format with the hotels and cities information. In our services, a parameter with the amount of records wanted must be sent. These records are limited to 1000 and a parameter with page number, in order to keep track of the information that is already downloaded. To download the static data, we recommend to download first all cities content and then all hotels content. When sending asynchronous request, it is recommended to wait until a response for your first request is received before sending the second request. We also will inform about important changes in the static data by email and it is advised to update that specific item immediately.

Get Cities

Service to execute data static to cities.

HTTP Request

POST http://apidev.shekinapi.com:8001/cities

URL Parameters

The following should be sent in the raw of the URL

Parameter Description
User User authenticated in the API
Password Password authenticated in the API
Token Token authenticated in the API
Cant Amount of records that will be consumed per page
Pag Number of page

Response Parameters

Parameter Description
Total quantity Quantity of registers that you must download
pages Quantity of pages that you must consult
result Response information in JSON format
cod_country Country code
code City code
country Country name
name City name
state State name

Get Hotels

Service to consume data static to hotel.

HTTP Request

POST http://apidev.shekinapi.com:8001/hotels

URL Parameters

The following should be sent in the raw of the URL

Request Parameters

Parameter Description
User User authenticated in the API
Password Password authenticated in the API
Token Token authenticated in the API
Cant Amount of records that will be consumed per page
Pag Number of page

Response Parameters

Parameter Description
Total quantity Quantity of registers that you must download
pages Quantity of pages that you must consult
result Response information in JSON format
cityCode City code

hotelCode Hotel code
hotelInfo Hotel information
address Hotel Address
amenities Something, such as a swimming pool or shopping centre, that is intended to make life more pleasant or comfortable for the people in a property
description Basic description of the hotel
hotelName Hotel Name
images Images of the hotel
latitude Latitude where the hotel is spatially located
longitude Longitude where the hotel is spatially located
stars Number of the category of the hotel
propertyCategory A property category represents the type of accommodation being offered, "to see the list of Types, check the Appendix Property Category"

Get Hotel Detail

Service to search data of a specific hotel .

HTTP Request

POST http://apidev.shekinapi.com:8001/hotelsdetails

URL Parameters

The following should be sent in the raw of the URL

Request Parameters

Parameter Description
User User authenticated in the API
Pass Password authenticated in the API
Token Token authenticated in the API
HotelCode Code of hotel

Response Parameters

Parameter Description
Total quantity number of hotels
pages page by hotel
result General information of hotel
cityCode City code
cityName City Name
hotelCode Code of hotel
hotelInfo General items of hotel
address Address name's hotel
amenities Amenities of hotel
description Language hotel
email Email's hotel
fax Number of hotel
hotelName Name's hotel
images Images of hotel
latitude coordinates
longitude coordinates
phones Phone number of hotel
stars Star count
web
zipCode zipCode
propertyCategory type of hotel (apartment, house, cabins)
