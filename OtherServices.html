Get Booking List

This Service shows all the existing bookings

HTTP Request

POST http://apidev.shekinapi.com:8001/bookinglist

URL Parameters

The following should be sent in the raw of the URL

Parameter Description
user User authenticated in the API
pass Password authenticated in the API
token Token authenticated in the API 
                                                                  

                                                                       
 The Request when sending in body (form-data) key="data" and value = JSON with the following structure:


                                                                        {
                                                                            "user" : "user",
                                                                            "pass" : "6e9552c9bd8e61c8f277c21220160234",
                                                                            "token" : "c270d866f86d10a235dbd634e1c7bc56",
                                                                            "filters" : {
                                                                                "locator" : [],
                                                                                "reference_client" : [],
                                                                                "createDate" : {
                                                                                    "from" : "",
                                                                                    "to" : "" 
                                                                                },
                                                                                "reservationDate" : {
                                                                                    "from" : "",
                                                                                    "to" : "" 
                                                                                },
                                                                                "status" : [],
                                                                                "currency" : []
                                                                            }
                                                                        }

Filters

An advance search can be made with the filters. We have 2 types of filters: Unique ones and the mixed ones.

Unique Filters

Parameter Descrption
locator Searches by booking code(s). If sent more than one, sort each with “,” for example
"locator" : ["BKHE-DC2915","BKHE-DC2916","BKHE-DC2917"]
reference_client Searches by the the agency reference(s). If sent more than one, sort each with “,” for example
"reference_client" : ["YASE1255","YASE1256"]

Filters can be mixed

With this type of filters, it can be sent all the filters, more than one or just one filter

Parameter Description
createDate Searches by the check-in date. Use “From” for the start date and “to” for the end date of the booking
reservationDate Searches by creation date. Use “from” the start date and “to” for the end date
status Searches by the status of the reservation. To see the list of the status, check the Appendix Status Booking
currency Searches by currency 
                                                                  
                                                                        
  The answer shows a JSON code with the following structure:


                                                                        {
                                                                            "booking_list": 
                                                                            [
                                                                                {
                                                                                    "checkIn": "2019-05-01",
                                                                                    "currency": "USD",
                                                                                    "locator": "BKHE-DC2915",
                                                                                    "reference_client": "YASE1255",
                                                                                    "reserve_date": "2019-04-09",
                                                                                    "status": 6,
                                                                                    "total_price": 475.64
                                                                                }
                                                                            ]
                                                                        }

Response Parameters

Parameter Description
checkIn Check in date
currency Reservation’s currency
locator Booking code
reference_client Client’s booking code
reserve_date Booking creation date
status Reservation status
total_price Total reservation price

Get Booking Detail

This Service shows reservation details

HTTP Request

POST http://apidev.shekinapi.com:8001/bookingdetails

URL Parameters

The following should be sent in the raw of the URL

Parameter Description
user User authenticated in the API
pass Password authenticated in the API
token Token authenticated in the API
locator Reservation reference 
                                                                  
                                                                      
 The Request when sending in body (form-data) key="data" and value = JSON with the following structure:
 

                                                                        {
                                                                            "user" : "user",
                                                                            "pass" : "6e9552c9bd8e61c8f277c21220160234",
                                                                            "token" : "c270d866f86d10a235dbd634e1c7bc56",
                                                                            "locator" : ""
                                                                        }

Response Parameters

Paramater Parameter Description
cancel_policies
description "amount" -> "baseType" Type which must be used to charge the penalty. To see the list of the types, check the Appendix Base Type Cancel Policies
description "amount" -> "number" Value that must be charged in the cancellation policy
description "time" -> "number" Number that must be applied in the cancellation policy
description "time" -> "type" It indicates if it is before or after the check-in date
description "time" -> "unit" It indicates how much must be discounted in the cancellation policy
details_hotel
checkIn Start date of the booking
checkOut End date of the booking
hotelCode Hotel code
name Hotel Name
info_invoice_and_cost
total_price Reservation’s total price
observations Reservation’s remarks
passengers_info address Passenger’s address
childAge Passenger’s age
city_billing Passenger’s city
country_billing Passenger’s country
documentNumber Passenger’s ID
email Passenger’s email
lastName Passenger’s last name
name Passenger Name
rph Guest room number
state Passenger’s province
typePass Passenger’s type. “10” if Adult, “8” If child
zipcode Passenger’s postal code
status Status of the reservation. To see the list or status, check Appendix Status Booking
