Availability

The code is sent using the standard OTA with the search parameters synchronously, which throws a transaction code and this must be sent again with the search parameters to obtain the availability list, in order to make it work faster. If "True" is sent in the SequenceNmbr parameter, a room per hotel will return in the response. This is not recommended because the response size increase the data transfer times and the quantity of product decrease in the response due to Time Out and data processing. To obtain a better performance use "False", whose response includes hotel data and the most economical value. When data based on hotel is required, the following service includes the entire stock of rooms.

TYPES OF SEARCH:

There are three types of search.

Search availability based on area:

It returns all the hotels according to the selected zone. When located in the node “HotelRef”, add these attributes: “HotelCityCode” : “ ”. It indicates the city code in which the hotel is situated. For example: HotelCityCode : ”54462254”

Search availability based on hotel:

It returns the hotel, the one selected. When located in the node “HotelRef”, add these attributes: “HotelCode” : “ ”. It indicates the hotel code.

Search with or without rooms:

It returns the rooms per hotel. Use the attribute: “SequenceNmbr” : “ “. For example: HotelCode : ”40862254”
  • True: returns data with bedrooms
  • False: returns data without bedrooms

Check prices

This service allows to check the rooms prices returned from the selected hotel. Each one of the returned rooms is identified with the RPH code, which will be used in the MoreDataEchoToken attribute

Considering the above, the correct steps to procede to make the respective check prices search are:

  • Select or chose a hotel and a specific package based on the result of the availability.
  • Select the specific package

Search and selection of the hotel:

Assuming that the research was made based on zone in the “Availability”, this would return a list of all hotels by zone, with their basic information such as (location, code or ID, and name). At this point, please select the values HotelCode y Brandcode.

To verify prices the values, that were previously returned in the response of the Availability, will be placed in the attributes HotelCode y BrandCode.

Select specific Package of rooms:

To select a room an important attribute must be used, which is returned in advance in the search’s response based on hotel. This is called “MarketCode”, which represents the identifier of one or more rooms according to the search of a hotel.

Note: Use in the request the returned value of the MarketCode attribute in the value of MoreDataEchoToken attribute.
